# Multi-touch planning (bounded)

Multi-touch planning suggests a **lightweight sequence** for working an account without generating spammy cadences.

## What it produces

- first touch suggestion (timing-led)
- second touch suggestion (value artifact + tight CTA)
- fallback routing guidance
- optional wait/monitor guidance when evidence is thin

## What it does not claim

- not a guaranteed best sequence
- not an automated sequencer
- not a “send now” capability

## API

- `GET /api/accounts/:id/touch-plan`
