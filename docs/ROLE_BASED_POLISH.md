# Role-based polish model

LeadIntel supports multiple roles without forking the product experience.

## Roles

- **Rep**: needs a tight daily loop (shortlist → why now → draft → action).
- **Manager**: needs consistency and rollout confidence across reps.
- **Operator**: needs governance, safe packaging, exports/webhooks, and auditability.
- **Admin**: needs configuration and operational visibility.

## Principles

- No separate “apps” per role.
- Subtle ordering and emphasis changes are allowed (CTA ordering, default tabs).
- Role-aware copy must remain truthful and map to real product objects.

## Examples (allowed)

- Operators see export/webhook actions higher in the action center.
- Reps see “copy opener” and “generate variants” higher.
- Managers see template governance calls-to-action when on Team plan.
