# Shell and navigation guidelines

## Principles
- **Primary actions stay primary**: limit the always-visible nav to a small set.
- **Group power-user surfaces**: team/ops/admin routes live behind “More” or role gates.
- **Starter is focused**: don’t overwhelm preview users with advanced destinations.

## Logged-in header (desktop)
- Always visible:
  - Dashboard
  - Actions
  - Reports
  - Pricing/Upgrade
- “More”:
  - Tour
  - Email preferences
  - Learn
  - Team/ops surfaces (Benchmarks, Partner, Rollouts, Ops) when tier allows

## Logged-in header (mobile)
- Keep the grid compact and plan-aware.
- Include Reports + Pricing for all tiers.
- Show Team/ops only when tier allows.
