# Page hierarchy guide (public + shell)

This guide standardizes how pages should read at a glance: spacing, headings, and CTA grouping. It is intentionally lightweight.

## Defaults
- **One clear H1** per page.
- **Section rhythm**: use consistent vertical spacing between major sections; avoid “cliffs” where one section is dense and the next is empty.
- **Dividers**: if a page needs separators, prefer subtle borders (`border-border/30`) over loud colored rules.

## Headings
- **H1**: strong, crisp, not overly decorative.
- **H2**: consistent size/weight across public pages (avoid random jumps).
- **Avoid neon everywhere**: neon headings should be the exception (hero moments), not the baseline.

## CTAs
- **Primary CTA**: one per section. Use glow/neon for the single main action.
- **Secondary CTA group**: 1–3 actions max; use `outline`/`secondary` styles.
- **Microtrust**: small text near decision points; keep it short and operational.

## Card composition
- **Title**: one clear line.
- **Supporting line**: one sentence max.
- **Action/state**: one button or one state block (locked/unavailable/loading).

## Responsive expectations
- No page-wide horizontal scroll.
- CTA groups should stack cleanly on small widths.
- Grids should drop to one column before cards feel cramped.
