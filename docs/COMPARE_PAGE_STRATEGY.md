# Compare page strategy (public)

## Goals

Compare pages should:

- help buyers self-qualify quickly
- stay conservative when details vary by plan/configuration
- avoid competitor-bashing and fabricated claims
- highlight LeadIntel’s differentiation: **why-now → shortlist → explain → draft → action**

## Required sections (each compare page)

- Hero + quick verdict
- Who each product is best for
- Where LeadIntel is better
- Where competitor is stronger
- Use together (when appropriate)
- Feature comparison table (truthful, conservative)
- Final recommendation based on buyer type

## Claim policy

- Prefer exact, workflow-level claims that LeadIntel implements.
- If uncertain, use “Varies by plan / configuration” instead of guessing.
- Never claim compliance certifications or integrations that aren’t implemented.
